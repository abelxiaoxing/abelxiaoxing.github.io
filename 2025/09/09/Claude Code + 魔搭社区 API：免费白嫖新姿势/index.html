Claude Code + 魔搭社区 API：免费白嫖新姿势

最近发现魔搭社区（ModelScope）已经支持 Anthropic API，这意味着我们可以通过一些简单的配置，让 Claude Code 直接使用魔搭社区提供的免费 API 资源。亲测可用，这里给大家分享一下具体步骤。

核心思路

核心原理非常简单：通过修改环境变量，将 Claude Code 的 API 请求地址指向魔搭社区的 API 端点。

魔搭社区API

  1. 打开魔搭社区：https://modelscope.cn/models
  2. 筛选可以api调用的模型：点击支持体验–>推理 APl-Inference,支持通过标准推理 AP接口迅速调用
  3. 点击推理 API-Inference下的Anthropic查看代码示例，复制自己的apikey和对应的模型ID。

claude code安装和配置

claude code安装

  1. 安装pnpm
    推荐使用 pnpm 而不推荐 npm，因为 pnpm 的安装速度更快，并且通过符号链接的方式能更有效地管理依赖，占用磁盘空间更小。关于 pnpm 的具体安装过程这里不再赘述，可以参考其官方文档进行安装。
  2. 安装claude code

模型选择与使用额度

  • 上面的示例默认使用了 ZhipuAI/GLM-4.5和Qwen/Qwen3-Coder-480B-A35B-Instruct 模型，该模型每天有 500次 的免费调用额度。
  • 魔搭社区总共每天提供 2000次 的免费调用额度。为了充分利用额度，可以向上述配置一样将不同的模型搭配使用。例如，可以将 ANTHROPIC_SMALL_FAST_MODEL 更换为 deepseek-ai/DeepSeek-V3.1 或其他支持的模型。

工具使用

文档工具

  1. 查看官方文档(Context7 MCP)

    • resolve-library-id - 解析库名到 Context7 ID
    • get-library-docs - 获取最新官方文档

  2. 搜索真实代码(Grep MCP)

    • searchGitHub - 搜索 GitHub 上的实际使用案例

编写规范文档工具(spec workflow MCP)

编写需求和设计文档时使用 specs-workflow：

  1. 检查进度: action.type="check"
  2. 初始化: action.type="init"
  3. 更新任务: action.type="complete_task"
